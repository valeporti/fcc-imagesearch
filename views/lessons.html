Lesson 1 - how does callbacks works

Pretty interesting to explain: The simplest way to undesrtand them, on my way of view is:

When creating a function and needing to do some stuff in a "synchronous" way, it's a good idea to put a "callback" variable at the end (it has not to be declared) that will work as the pivot that you build meant to order the function when to end the activity for which you built it and then continue the part of code that is inside the external function. Let me explain with an example, hope it helps: 

      --------------- Function declaration (let it be in the scripts.js file for example) --------------
          function doSomething (variable1, variable2, callback) {
            if ((variable2 * -1) < 0) then {
              setTimeOut(function() {
                cb ("variable2 is positive");
              }, 3000);
            } 
            else {
              var doSomeAwesomeness = variable1 * variable 2;
              cb (doSomeAwesomeness);
            }
          }
          
      --------------- Function usage (let it be in the main.js file for example) ------------------------
          [... something before ...]
      
          doSomething (1, 2, function(stringReturned) {
            CODE TO DO AFTER CALLBACK
            //we should expect that because 2 > 0, until 3 seconds later, this code will actually run (because of the "setTimeOut")
          }
      
          [... something after ...]
